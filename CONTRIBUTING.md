# Contributing to AI Transformation Lab

Thank you for your interest in improving this program. Contributions of all kinds are welcome — from fixing a typo to adding a new session exercise.

---

## Ways to contribute

**Content improvements**
- Better worked examples for specific functions (Finance, Supply Chain, Strategy, Ops, HR, Legal, etc.)
- Stronger "Do this now" prompts
- Additional prompt patterns for the By Function tab
- Improvements to the guardrails section for specific industries

**Technical improvements**
- Bug fixes
- Mobile experience improvements
- Accessibility improvements
- Performance improvements

**Program design**
- New session structures or formats
- Ideas for the builder track
- Suggestions for the self-paced mode

---

## How to contribute

1. **Fork** this repository
2. **Create a branch** for your change: `git checkout -b your-change-description`
3. **Make your changes** to `index.html` (everything is in one file)
4. **Test** by opening the file in a browser and checking your changes work
5. **Submit a pull request** with a clear description of what you changed and why

---

## Guidelines

**Keep it tool-agnostic.** The program intentionally works with any AI tool. Contributions should not favour or require a specific AI provider.

**Keep it self-contained.** Everything must remain in a single HTML file with no external dependencies beyond Google Fonts. No build steps, no npm, no frameworks.

**Keep it accessible.** The program is designed for professionals across all levels and functions — not developers. Content should be clear to someone who has used AI tools casually but wants to go deeper.

**Test on mobile.** A significant portion of users will first encounter this on a phone. Any changes should be checked at mobile viewport widths.

**Respect the structure.** Sessions 1–3 are the core program. Session 4 is the builder track. The Prompt Lab, Wins Wall, Guardrails, and Connect tabs are tools and reference. Changes that restructure this need strong justification.

---

## Reporting issues

Use GitHub Issues to report bugs or suggest improvements. Please include:
- What you were doing when the issue occurred
- What browser and device you were using
- A screenshot if relevant
